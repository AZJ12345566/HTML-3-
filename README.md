# HTML-3-
HTML 属性，标题，段落
//HTML可以拥有属性，属性提供了有关HTML元素的更多信息
//属性总是以名称/值对的形式出现，如：name=“value”
//属性总是在HTML元素的开始标签中规定的
//例：
    <a href="http://www.wschool.com.cn">This is a link</a>
    //这里的HTML链接由<a>标签定义。链接的地址在href属性中指定

//例：
    <h1 align="center">
    //这里拥有关于对齐方式的附加信息
      
//例：
    <body bgcolor="yellow">
    //这里拥有关于背景颜色的附加信息
      
//例：
    <table border="1">
    //拥有关于表格边框的附加信息
      
//始终为属性值加引号
  //属性值应该始终被包括在引号内。双引号最常见，不过使用单引号也没问题
  //在某些特殊的情况下，比如属性值本身就含有双引号，那么就必须使用单引号了
  name='Bill"HelloWorld"Gates'


//HTML标题
      //标题是通过<h1>-<h6>等标签来进行定义的
      //<h1>为最大标题<h6>为最小标题
      
      //HTML水平线
      //<hr>标签在HTML页面中创建水平线
      //hr 元素可用于分隔内容
      <p>This is paragraph</p>
      <hr />
      <p>This is paragraph</p>
      //使用水平线<hr标签>来分隔文章中的小节是一个办法（但不是唯一的办法）
      
      //HTML注释
      <!--This is a comment-->
      //格式：开始括号之后(左边的括号)需要紧跟一个感叹号，而右括号则无需添加括号
      
      
//HTML段落
      //HTML段落是通过<p>标签来定义的
      //注：浏览器会自动在段落的前后添加空行(<p>是块级元素)
      //<p></p>去插入空行是一个坏习惯。<br />标签去代替它
      
      //HTML折行
      //若果在不产生一个新的段落的情况下进行换行(新行),使用<br />标签：
      <p>This is<br />a para<br />graph with line breaks</p>
      //<br />元素是一个空HTML元素，由于关闭标签没有任何意义，因此他没有结束标签
      //<br>与<br />：即使<br>在所有浏览器中的显示都没问题，使用<br />也是更长远的保障
      
      //HTML输出-有用的提示
      //我们无法确定HTML被显示的确切效果。屏幕的大小，以及对窗口的调整都可能导致不同的结果
      //我们无法通过在HTML代码中添加额外的空格或换行来改变输出的结果
      //在显示页面时，浏览器会自动移除源代码中的多余的空格和空行，连续的空格和空行会被视为一个空格
